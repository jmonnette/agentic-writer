# Agentic Writing System

A modular, multi-agent pipeline for generating high-fidelity articles with consistent voice and stance — built for Claude Code.

## Quick Start

```bash
# 1. Clone and initialize
git clone <this-repo>
cd agentic_writer
chmod +x setup.sh && ./setup.sh

# 2. Add your existing writing samples
cp your_articles/* library/

# 3. Index your library and calibrate your voice
# In Claude Code:
# "Run the Archivist on the library"
# "Run the Stylist to calibrate my voice"

# 4. Start writing
# "Research [your topic]"
```

## System Architecture

### Knowledge Core (Memory)
- **Archivist**: Indexes `/library` content into `LIBRARY_INDEX.md`; indexes third-party source material into `EXTERNAL_LIBRARY_INDEX.md`
- **Librarian**: Generates POV Briefs from the index for cross-article consistency
- **Stylist**: Analyzes your writing patterns to build a personal voice profile

### Production Team (Execution)
- **Interviewer**: Intake agent — sharpens angle, audience, and intent before research begins
- **Researcher**: Deep-dive web research using Brave Search; auto-indexes results into `RESEARCH_INDEX.md`
- **Ghostwriter**: Transforms research + POV into structured drafts
- **Critic**: Audits logic, evidence gaps, and argument strength
- **Persona Reviewer**: Tests article from audience perspectives
- **Editor**: Polishes for publication (15% brevity rule)
- **Social Media Writer**: Generates LinkedIn and X/Twitter posts from finished articles

### Persona System (Audience Validation)
- **Persona Generator**: Creates custom audience profiles on demand
- **Persona Reviewer**: Evaluates drafts from specific persona perspectives
- **Persona Library**: Pre-defined personas + custom ones in `.claude/personas/`

## Full Article Pipeline

Each article gets a dated subfolder: `drafts/[topic]_YYYYMMDD/`

1. **Interview** (recommended)
   ```
   "Interview me about [topic]"
   → Outputs: drafts/[topic]_YYYYMMDD/interview_brief.md
   ```

2. **Research**
   ```
   "Research [topic]"
   → Outputs: drafts/[topic]_YYYYMMDD/research_pack.md
   ```

3. **POV Brief**
   ```
   "Generate a POV brief for [topic]"
   → Outputs: drafts/[topic]_YYYYMMDD/pov_brief.md
   ```

4. **Outline**
   ```
   "Create an outline for [topic]"
   → Outputs: drafts/[topic]_YYYYMMDD/outline.md
   ```

5. **Draft**
   ```
   "Draft the article"
   → Outputs: drafts/[topic]_YYYYMMDD/v1.md
   ```

6. **Critique**
   ```
   "Critique the draft"
   → Outputs: drafts/[topic]_YYYYMMDD/critique.md
   ```

7. **Revise**
   ```
   "Revise based on the critique"
   → Outputs: drafts/[topic]_YYYYMMDD/v2.md
   ```

8. **Persona Review** (optional)
   ```
   "Review from enterprise-cto perspective"
   → Outputs: drafts/[topic]_YYYYMMDD/persona_review_enterprise-cto.md
   ```

9. **Final Edit**
   ```
   "Edit to final"
   → Outputs: drafts/[topic]_YYYYMMDD/FINAL.md
   ```

10. **Social Posts** (optional)
    ```
    "Write social posts for this article"
    → Outputs: drafts/[topic]_YYYYMMDD/social_posts.md
    ```

## Quality Standards

1. **POV Consistency** — Librarian ensures stances align with your established positions
2. **Argument Strength** — Critic checks logic, evidence, and counterarguments
3. **Voice Alignment** — Editor verifies match with your personal voice profile
4. **Brevity Target** — 15% reduction from first draft (Editor phase)
5. **Evidence Standards** — All claims require citation or logical support
6. **Attribution** — Third-party source material is cited with author, publication, and date

## Project Philosophy

Three core separations prevent context drift and ensure quality:

1. **Content vs. Form** — What is said vs. how it's said
2. **Logic vs. Polish** — Bones of argument vs. skin of prose
3. **Scouting vs. Synthesis** — Gathering data vs. crafting narrative

Each agent has a single mandate. That focus is what makes the output good.
